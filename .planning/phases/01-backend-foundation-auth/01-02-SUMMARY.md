## Decisions Made
- **Direct bcrypt over passlib:** RESEARCH.md confirmed passlib 1.7.4 is incompatible with bcrypt 5.0.0. Used direct bcrypt calls per verified runtime compatibility.
- **Alembic sync URL:** Alembic is sync-only. Used `sqlite:///../data/dev.db` in alembic.ini for migration operations while app uses async `sqlite+aiosqlite:///./data/dev.db` from config.
- **User model import in env.py:** Required to populate Base.metadata for autogenerate detection of the users table.
- **CORS config:** Used comma-separated env var (CORS_ORIGINS) parsed into list, with allow_credentials=True (required for cookie-based auth).

- Seed script must be run from project root (not backend/ directory) because module resolution requires the hellojobs root on sys.path.
